- Reason: `CategoryBreakdown` does not consume `categories` (uses `row.name`/`row.color` directly); passing it would be an unused prop that lint would flag

## CategoryBreakdown `categories` prop decision

**Removed.** `byCategory(items, categories)` (in `asset-compute.ts`) already resolves `name` and `color` before returning `CategoryBreakdown[]`. So the component only needs `cats: CategoryBreakdown[]` — no second-pass lookup. Adding an unused prop would violate the no-unused-vars lint rule.
