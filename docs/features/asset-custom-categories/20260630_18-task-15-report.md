## No-Loop Re-fetch Guard

`load()` 호출당 `hasLegacy` 체크를 수행하고, `migrateExpenseCategories` 호출과 재조회는 **최대 1회**만 실행된다.
- `hasLegacy = expM.some((e) => e.categoryId === null)` 로 확인.
- migrate 후 재조회한 `freshExpM` 에는 이미 categoryId 가 채워지므로, 다음 `load()` 호출 시 `hasLegacy = false` → migrate 건너뜀.
- 루프 구조가 없으므로 무한 재귀 불가.
