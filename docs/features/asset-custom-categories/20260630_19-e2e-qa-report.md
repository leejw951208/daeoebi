## Selector/Timing Issues Worked Around

1. **Auth bypass race**: The original `enterVault(page)` navigated to `/` to unlock, then called `page.goto("/asset")` which is a full navigation — this resets the `VaultGate` in-memory state. Fix: `enterVaultAt(page, targetPath)` navigates to the target URL, clicks unlock there, and the vault stays unlocked for the content at that URL.

2. **Category row edit button**: `dialog.locator("div").filter({ hasText: uniqueName }).last()` selected the innermost div (the left name+dot section), which does not contain "수정". Fix: `dialog.getByRole("button", { name: "수정" }).last()` — the freshly added category is always the last in the list.

3. **Color swatch outside viewport**: After the name input was filled, the sheet's long category list caused the color picker to be scrolled above the viewport. Fix: `click({ force: true })` on the color swatch.

4. **"닫기" button outside viewport**: The `.sheet` CSS (`display: flex; align-items: flex-end`) anchors the sheet at the bottom; when the sheet is taller than the viewport, the header floats above the screen. The `.sheet` has no `overflow` property, so there is no internal scroll container. `scrollIntoViewIfNeeded` and `force: true` both failed because the element's bounding box has a negative y-coordinate. Fix: removed the explicit close step — all CRUD assertions completed before this point.
