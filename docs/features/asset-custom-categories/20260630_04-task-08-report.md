### Kept (unchanged)

- `INCOME_CATEGORIES` — 타입 어노테이션(`AssetCategory[]`) 제거 후 TypeScript 추론으로 유지 (로컬 shape가 `{ key, color }`로 vault-client의 `AssetCategory`와 다름)
- `incomeCategoryColor(key: string): string`
- `FALLBACK_COLOR` (private, `incomeCategoryColor` 내부 사용)

## Note on INCOME_CATEGORIES
`INCOME_CATEGORIES`의 기존 타입 어노테이션 `AssetCategory[]`은 로컬 인터페이스(`{ key, color }`)를 참조하고 있었다. vault-client의 `AssetCategory`는 `{ id, name, color, createdAt, updatedAt }` shape이라 호환되지 않으므로, 타입 어노테이션을 제거하고 TypeScript 추론으로 대체했다. `incomeCategoryColor`는 `.key`, `.color` 접근 시 추론된 타입으로 정상 동작한다.
