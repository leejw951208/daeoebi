## Component Structure

`CategoryManager.tsx` contains four internal units kept in a single file:

1. **`ColorPicker`** — 10-swatch palette grid (`CATEGORY_PALETTE`). Each swatch is a circular `<button>` with `aria-pressed` for accessibility. Active swatch highlighted by border.

2. **`AddSection`** — Controlled form (name input + `ColorPicker`). Manages own `name`/`color`/`saving` state; calls `onAdd(name, color)` prop and resets on success.

3. **`CategoryRow`** — Per-category row with view and edit modes. In view mode: color dot + name + 수정/삭제 text buttons. In edit mode: name input + `ColorPicker` + 저장/취소 buttons. Manages own `editing`/`saving` state; calls `onEdit(id, patch)` on save.

4. **`CategoryManager`** (main export) — Owns the category list state, loads on mount via `listAssetCategories()`, handles add/edit/delete via vault-client functions, shows `ConfirmDialog` for deletes, displays errors via `isApiError` → `error-box`. Calls `onChanged?.()` after every successful write.

File size: ~290 lines (within 400-line budget).
