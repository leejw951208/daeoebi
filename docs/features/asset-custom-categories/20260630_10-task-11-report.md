# Task 11 Report: 카테고리 로드 + categoryId 빌드 + 머티리얼라이즈 전달

## Files Changed

### 1. `apps/web/app/(vault)/asset/page.tsx`
- Added `listAssetCategories` to the import from `@/lib/vault-client` (no `AssetCategory` type needed here — type is inferred from return value; explicit import was unused and rejected by ESLint).
- Extended `Promise.all` destructuring to capture `categories: AssetCategory[]` as the fourth element.
- Changed `ComputedExpense` build: replaced `category: p.category` (encrypted payload field) with `categoryId: v.categoryId` (plaintext FK from `ExpenseView`).
- Added `categories` to `setState({ status: "ready", data: { incomeAmount, incomes, expenses, categories } })`.

### 2. `apps/web/app/(vault)/asset/_lib/asset-recurring.ts`
- Added `categoryId: t.categoryId ?? undefined` to the `createExpense({...})` call inside `materializeRecurring`. `RecurringView.categoryId` is `string | null`; `?? undefined` converts null → undefined to match `CreateExpenseInput.categoryId?: string`.

### 3. `apps/web/app/(vault)/asset/_components/dashboard/AssetDashboard.tsx`
- Imported `type AssetCategory` from `@/lib/vault-client`.
- Added `categories: AssetCategory[]` to the `Loaded` interface (this is the shared data shape consumed by `page.tsx` setState and `AssetDashboard`).
- Changed `byCategory(data.expenses)` → `byCategory(data.expenses, data.categories)` (matches updated signature in `asset-compute.ts`).
- Added `categories={data.categories}` prop to `<CategoryBreakdown>` and `<DayDetail>` (Task 12/13 will add these to those components' Props interfaces).
