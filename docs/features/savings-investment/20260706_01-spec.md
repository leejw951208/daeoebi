# 저축·투자 관리 — 설계 (Spec)

- 작성일: 2026-07-06
- 대상: 자산 대시보드에 "저축·투자" 탭 추가 (디자인 `daeoebi.dc.html`)
- 브랜치: `feat/asset-savings-investment` (develop 기준)

## 1. 목적·범위

자산 대시보드에 **"저축·투자" 뷰**를 추가한다. 상단 세그먼트로 `이번 달`(기존 예산·지출)과 `저축·투자`를 전환한다. 저축·투자 뷰는 **기존 지출을 연동한 파생 뷰**로, 저축·투자 순자산과 이번 달 적립 내역, 저축 목표 진행률을 보여준다.

**핵심 결정 (합의됨)**
- 저축·투자는 **기존 지출 중 카테고리가 `저축`/`투자`인 건을 집계한 파생 뷰**다. 별도 계좌·잔액 입력은 없다.
- `저축`과 `투자`는 **각각 분리 집계**한다(카드·합계 별도).
- 저축 목표는 **저축만 단일 목표**(이름 + 금액)를 둔다. 투자 목표는 없다.
- 저축·투자 지출은 **기존대로 지출·남은 예산에 그대로 포함**한다(비파괴). 저축·투자 탭은 같은 데이터의 다른 뷰일 뿐이다.

**비범위(YAGNI)**: 투자 평가액·수익률, 저축/투자 계좌 CRUD, 다중 목표, 목표 알림.

## 2. 데이터 모델

- **`SavingsGoal`** (신규, 싱글톤 단일 행)
  - `name: String` — 목표 이름(평문 라벨)
  - `iv/ciphertext/authTag: Bytes` — 암호화된 `{ amount }` 블롭 (VK, AES-256-GCM). 서버는 복호화하지 않는다(Income·Secret 패스스루와 동일).
  - `createdAt/updatedAt`
  - 단일 행: 서비스가 "행이 있으면 update, 없으면 create"로 싱글톤을 강제한다.
- **저축/투자 카테고리**: 기존 `AssetCategory`(시드 이름 `저축`·`투자`)를 그대로 사용. 별도 저장 없음. 식별은 클라이언트가 카테고리 목록에서 이름으로 판별한다(이름 변경 시 해당 집계에서 빠짐 — v1 허용).

새 migration으로 `SavingsGoal` 테이블 생성.

## 3. 백엔드 (NestJS `asset` 모듈)

Income CRUD의 **암호문 패스스루 패턴**을 그대로 따른다(base64url 인·디코드, `toView`).

- **SavingsGoal**
  - `savings-goal.service.ts`: `get()` → 행 또는 `null`, `upsert(dto)` → 행 있으면 update, 없으면 create.
  - `savings-goal.controller.ts`: `GET /savings-goal`, `PUT /savings-goal`.
  - `dto/savings-goal.dto.ts`: `name`(1~40자) + base64url `iv`·`ciphertext`·`authTag`.
- **Expense 확장**
  - `listContributions(categoryIds: string[])` — 전 기간 `removed=false` 이고 `categoryId ∈ ids` 인 지출을 블롭 포함 `ExpenseView[]`로 반환. `categoryId` 인덱스 사용.
  - controller: `GET /expenses/contributions?categoryIds=a,b` (빈 목록이면 빈 배열).
- 모듈에 컨트롤러·서비스 등록 + CsrfMiddleware 적용.

## 4. 계산 (`asset-compute` 순수 함수)

기존 `asset-compute.ts`에 순수 함수로 추가한다(서버 복호화 없음 — 클라가 복호화한 값으로 집계).

- `savingsSummary(contribs, categories)` → `{ savedTotal, investTotal, netWorth }`
  - `contribs`: 복호화된 적립 지출 `{ categoryId, amount }[]` (전 기간)
  - 카테고리 이름이 `저축`이면 `savedTotal`, `투자`면 `investTotal`에 합산. `netWorth = savedTotal + investTotal`.
- 월별: 현재 달 지출에서 저축/투자를 필터해 `savedMonth`·`investMonth`·적립 리스트 산출(기존 `byCategory`/월 데이터 재사용).
- `goalProgress(savedTotal, goalAmount)` → 정수 pct(0~100 클램프, `goalAmount<=0`이면 0).

## 5. 프론트 클라이언트·암호화

- `vault-client.ts`
  - `SavingsGoalView { id, name, iv, ciphertext, authTag }`
  - `getSavingsGoal(): Promise<SavingsGoalView | null>`
  - `saveSavingsGoal(name, blob): Promise<SavingsGoalView>` (PUT)
  - `listContributions(categoryIds: string[]): Promise<ExpenseView[]>`
- `asset-payload.ts`: `sealGoal(vk, { amount })` / `openGoal(vk, blob)` — 기존 `seal`/`open` 재사용.

## 6. UI

- **`AssetDashboard`**: 상단 세그먼트(`이번 달` ↔ `저축·투자`) 상태 추가. `이번 달`은 기존 렌더 그대로.
- **저축·투자 탭 컴포넌트**(신규): 순자산 hero + "이번 달 N원 적립" · 저축/투자 카드(각 누적+이번 달) · 저축 목표 진행바(`저축 목표 · {name}`, pct) · 이번 달 적립 내역 리스트(지출 상세로 링크).
- **`SavingsGoalSheet`**(신규): 목표 이름 + ₩ 금액 입력·저장. `BudgetSheet` 패턴(취소/저장, VK 암호화, 저장 실패 문구).
- **`asset/page.tsx`**: 저축·투자 탭 최초 진입 시 `listContributions`(저축/투자 categoryIds) + `getSavingsGoal` **지연 로드**. 세그먼트·목표 시트 상태 관리. 목표 저장·지출 변경 후 재로드.

## 7. 에러 처리

- 적립/목표 조회 실패 → "불러오지 못했습니다." (기존 자산 패턴)
- 목표 저장 실패 → "저장하지 못했어요. 다시 시도해 주세요." (BudgetSheet 문구 일치)
- 복호화 실패 항목은 스킵하고 나머지 정상 표시(기존 패턴).

## 8. 테스트

- **단위**
  - `asset-compute`: `savingsSummary`(저축/투자 분리·순자산), `goalProgress`(클램프·0 목표), 월별 필터.
  - `SavingsGoalService`: get(null/행), upsert(생성/수정 싱글톤), 암호문 패스스루.
  - `ExpenseService.listContributions`: categoryIds 필터·removed 제외·빈 목록.
- **e2e**: 세그먼트 전환, 순자산·저축/투자 카드 표시, 목표 설정→진행률 반영.
- **QA 시나리오**: `SAVINGS` 분류 케이스(탭 전환·목표 저장·적립 집계) 추가.

## 9. 비파괴 보장

기존 지출·예산·고정지출·카테고리 로직은 변경하지 않는다. 저축·투자는 기존 지출 데이터를 읽어 별도 뷰로 집계할 뿐이다. `SavingsGoal`·`listContributions`는 추가만 한다.
