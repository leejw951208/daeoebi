# UI 디자인 대조 (Claude Design ↔ 현재 구현)

Claude Design 프로토타입([daeoebi.dc.html](./design-original/daeoebi.dc.html))과 현재 웹앱(`apps/web`) 구현을 화면별로 대조한 자료.

- **원본**: `design-original/` — 디자인 프로토타입 원본(`daeoebi.dc.html`)과 렌더 런타임(`support.js`). `{{ }}`·`<sc-if>`·`<sc-for>`는 DC 템플릿 문법이며, 스타일은 인라인 `style`에 들어 있음.
- **차이 스니펫**: `diff/` — 디자인이 **추가되거나 현재 구현과 다른** 화면의 디자인 마크업을 그대로 복사하고, 상단 주석에 차이를 정리.

## diff/ 파일

디자인이 추가되거나 다른 부분의 디자인 코드(html/인라인 css):

- [`diff/01-onboarding.html`](./diff/01-onboarding.html) — 온보딩 문구·체크리스트·토큰
- [`diff/08-field-suggestions.html`](./diff/08-field-suggestions.html) — 앱에 없는 독립 "필드 추천" 화면
- [`diff/11-asset-dashboard.html`](./diff/11-asset-dashboard.html) — 수입/월급 모델(hero·수입 카드·월 수입 시트)
- [`diff/12-expense-form.html`](./diff/12-expense-form.html) — 결제방법 칩·카테고리·고정 토글

> 2·3·4·7·10번은 문구·구조가 거의 일치하여 별도 스니펫을 두지 않음(원본에서 확인 가능). 5·6·9번의 소차이는 위 표 참고.

## 상세 차이

### 1 · 온보딩 — 다름
- 헤드라인: 디자인 `비밀번호도 가계부도, 안전하게 잠가두고 필요할 때만 엽니다` ↔ 앱 `비밀번호를 안전하게 잠가두고, 필요할 때만 엽니다`
- eyebrow: 디자인 `암호화 대외비 · 자산 관리` ↔ 앱 `대외비`
- 체크: 디자인 3줄(본인 확인 / 수입·지출과 남는 돈 자동 정리 / 자동 재잠금) ↔ 앱 2줄(가계부 줄 없음)
- 부트스트랩 토큰 입력: 디자인 없음 / 앱 있음

### 11 · 자산 대시보드 — 다름 ★
디자인은 **월 수입(월급)** 을 입력받는 모델, 현재 앱은 `수입→월 예산 전환` 이후 **월 예산 단건** 모델.
- hero 라벨: `이번 달 남은 돈`(디자인) ↔ `이번 달 남은 예산`(앱)
- 카드: `수입 · 월급` + `월 수입 설정` 시트(디자인) ↔ `이번 달 예산` + `월 예산` 시트(앱)
- 시트 설명: `매달 들어오는 월급을 입력하면 남는 돈이 자동 계산`(디자인) ↔ `이번 달 쓸 수 있는 돈을 정해두세요.`(앱)
- 카테고리별 지출·지출 달력·선택일 상세·잠금 타이머·하단 탭바는 구조 동일.

### 12 · 지출 폼 — 다름 ★
- 결제방법 칩(`카드`/`자동이체`/`현금`): 디자인 있음 ↔ 앱 UI 없음(주석만)
- 카테고리: 디자인 6종(`식비`·`교통`·`주거·공과금`·`쇼핑`·`문화`·`기타`) ↔ 앱 시드 8종(`+저축`·`+투자`), 데모는 6종
- 고정 지출: 디자인 단순 토글 ↔ 앱 토글 + `개월 수 · 선택` 입력(비우면 무기한)

### 8 · 필드 추천 — 디자인에만
디자인은 그룹 카드 형태의 독립 화면(`필드 추천`)을 두지만, 앱은 별도 화면 없이 신규/수정 폼 1단계의 추천 칩(`field-suggestions.ts`)으로만 제공.

## 참고

- 디자인 프로토타입은 mood 테마 3종(ink/toss/carrot)을 프리뷰용으로 포함하나, 앱은 단일 테마.
- 공통 스타일 클래스(`asset-card`, `entry-card`, `sticky-header`, `fab`, `lock-timer` 등)는 `apps/web/app/globals.css`에 정의됨.
