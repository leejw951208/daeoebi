> 전체 기능 QA 시나리오. 각 케이스는 Playwright로 실행하고 **실제 결과**와 **Pass/Fail** 컬럼을 채운다.
> 실행 방법·원칙·기록 표기는 [README.md](./README.md) 참고. 작성 스킬: `.claude/skills/writing-qa-scenarios`.

## 인증 (AUTH)

| 분류 | 시나리오 ID | 사전 조건 | 절차 | 예상 결과 | 실제 결과 | Pass/Fail |
|------|-----------|---------|------|---------|---------|-----------|
| 인증 | AUTH-001 | 미등록 상태(온보딩 화면) | 1. `/` 접속 2. "패스키 만들기" 클릭 3. 생체/기기 인증 완료 | 패스키 등록 후 복구코드 발급 화면("복구코드를 저장하세요")으로 이동 |  |  |
| 인증 | AUTH-002 | 복구코드 발급 화면 | 1. 발급된 복구코드 확인 2. 저장/다음 진행 | 금고 목록("대외비 관리")으로 진입 |  |  |
| 인증 | AUTH-003 | 등록됨, 잠금 상태(잠금해제 화면) | 1. 패스키 잠금해제 시도 2. 생체 인증 완료 | 금고 목록으로 진입, 본문 열람 가능 |  |  |
| 인증 | AUTH-004 | 잠금 화면 | 1. "복구코드로 접근" 클릭 2. 잘못된 복구코드 입력 3. "복구코드 제출" | "복구코드가 일치하지 않습니다"(또는 형식 오류) 노출, 진입 실패 |  |  |
| 인증 | AUTH-005 | 잠금 화면 | 1. 복구코드를 5회 연속 오입력 | 5회째 이후 60초 잠금(백오프) 발동, 재시도 차단 안내 |  |  |
| 인증 | AUTH-006 | 잠금해제됨(자산/보관함 세션 활성) | 1. 3분간 무동작 유지 | 자동 잠금 발동 → 잠금 화면으로 전환 |  |  |
| 인증 | AUTH-007 | 세션 활성 | 1. 우상단 "m:ss 잠그기" 타이머 관찰 2. 타이머 버튼 탭 | 카운트다운 매초 갱신·60초 이하 urgent 강조, 탭 시 즉시 잠금 화면 |  |  |

| 시크릿 | SECRET-005 | 항목 상세 | 1. 삭제 2. 확인 다이얼로그 승인 | 목록에서 제거 |  |  |
| 시크릿 | SECRET-006 | 항목 추가/수정 폼 | 1. 필드 추가·추천 칩 사용 2. 순서 이동 3. 개별 필드 "삭제" | 각 동작 정상, 개별 삭제 시 나머지 필드는 유지 |  |  |
| 시크릿 | SECRET-007 | 항목 추가 폼 | 1. 필드 민감 토글 🔒/👁 전환 2. "저장 전 확인" 리뷰 확인 | 폼에서 마스킹 전환, 리뷰에서 민감 필드 마스킹 표시 |  |  |

| 분류 | 시나리오 ID | 사전 조건 | 절차 | 예상 결과 | 실제 결과 | Pass/Fail |
|------|-----------|---------|------|---------|---------|-----------|
| 가계부-예산 | BUDGET-001 | dev-unlock 상태, 자산 화면(`/asset`) | 1. "이번 달 예산" 카드("설정하기"/수정) 탭 | "월 예산" 바텀시트 열림(월 라벨 + "이번 달 쓸 수 있는 돈을 정해두세요.") |  |  |
| 가계부-예산 | BUDGET-002 | 월 예산 시트 | 1. ₩ 금액 입력 2. "저장" | 대시보드 "이번 달 예산"·"이번 달 남은 예산"에 반영 |  |  |
| 가계부-예산 | BUDGET-003 | 예산 설정됨 | 1. 예산 금액 변경 2. 저장 | 변경 금액이 대시보드에 반영 |  |  |
| 가계부-예산 | BUDGET-004 | 월 예산 시트 | 1. 금액 0/빈값 상태 | "저장" 버튼 비활성(저장 불가) |  |  |
| 가계부-예산 | BUDGET-005 | 월 예산 시트, API 실패 | 1. 저장 시도(서버 오류) | "저장하지 못했어요. 다시 시도해 주세요." 노출, 시트·입력 값 유지 |  |  |

## 가계부-카드결제월 (CARD)

> **카드** 지출은 구매월이 아니라 **익월(결제월)**에 반영된다.

| 분류 | 시나리오 ID | 사전 조건 | 절차 | 예상 결과 | 실제 결과 | Pass/Fail |
|------|-----------|---------|------|---------|---------|-----------|
| 가계부-카드결제월 | CARD-001 | dev-unlock 상태 | 1. 6월에 **카드** 지출 입력 2. 6월 화면 확인 | 6월 화면의 남은 예산/달력에 **반영되지 않음** |  |  |
| 가계부-카드결제월 | CARD-002 | CARD-001 이후 | 1. **7월 화면**(결제월)으로 이동 | 같은 카드 지출이 7월에 등장 |  |  |
| 가계부-카드결제월 | CARD-003 | dev-unlock 상태 | 1. 현금·자동이체 지출 입력 | 입력한 달(당월)에 그대로 반영 |  |  |
| 가계부-카드결제월 | CARD-004 | 카드 지출 존재 | 1. 결제월 달력 확인 | 카드 건이 결제월의 **같은 일자**(말일 클램프)에 달력 표시 |  |  |
| 가계부-카드결제월 | CARD-005 | 카드 지출 존재 | 1. 결제월 선택일 상세 확인 | 카드 건에 "M/D 구매" 부제 표시 |  |  |
| 가계부-카드결제월 | CARD-006 | 카드+비카드 지출 혼재 | 1. 지출 합계 vs 남은 예산 대조 | 지출 합계 = 남은 예산 차감 금액 일치 |  |  |
| 가계부-카드결제월 | CARD-007 | dev-unlock 상태 | 1. 카드 지출 1/31 구매 입력 2. 2월 결제월 확인 | 2월 결제는 **2/28(말일)**로 클램프 표시 |  |  |

## 가계부-고정지출 (RECURRING)

|------|-----------|---------|------|---------|---------|-----------|
| 가계부-고정지출 | RECURRING-001 | dev-unlock 상태 | 1. 지출 추가 폼 "고정 지출" 토글(신규만) ON 2. 저장 | 당월 인스턴스 생성 + "고정" 배지 |  |  |

| 가계부-고정지출 | RECURRING-003 | 비카드 고정, 6월 설정 | 1. 6월 화면 확인 2. 5월(이전 달)로 이동 | 6월부터 생성, **이전 달엔 생성 안 됨**(소급 없음) |  |  |

| 가계부-고정지출 | RECURRING-005 | 개월 수 3 + 6월 시작(비카드) | 1. 6·7·8·9월 확인 | 6·7·8월 생성, **9월엔 미생성** |  |  |
| 가계부-고정지출 | RECURRING-006 | 개월 수 3 + 6월 시작(카드) | 1. 7·8·9·10월 확인 | 결제월 7·8·9월 등장, **10월엔 없음** |  |  |
| 가계부-고정지출 | RECURRING-007 | 개월 수 비우고 생성 | 1. 먼 미래 달로 이동 | 무기한으로 계속 생성 |  |  |
| 가계부-고정지출 | RECURRING-008 | 고정 존재 | 1. 같은 달 재진입·새로고침 | 중복 인스턴스 안 생김(자동 생성 멱등) |  |  |
| 가계부-고정지출 | RECURRING-009 | 고정 인스턴스 수정 화면 | 1. 액션 노출 확인 | "고정 해제"(보조) + "삭제"(빨강) 둘 다 표시 |  |  |
